# Feature Specification: Autoffiliate Semi-Auto MVP

## 1. Overview
Autoffiliate is a semi-automated content management system designed for scaling TikTok Affiliate operations. It leverages human-assisted data sourcing from Kalodata and Gemini 2.0 for creative reasoning to produce high-quality, unique affiliate videos. Posting is handled manually to ensure account safety and optimal engagement.

## 2. Problem Statement
Fully automated scraping and posting are increasingly vulnerable to bot detection and shadowbans. Manual content creation, however, is too slow for scaling. A semi-auto approach balances safety with efficiency.

## 3. Goals & Objectives
- **Safety:** Minimize risk of bans by using human-assisted scraping and posting.
- **Scalability:** Enable rapid video generation from structured data (Excel/CSV).
- **Originality:** Ensure every video is unique through Gemini-powered scripting and asset randomization.
- **Flexibility:** Support multiple niches via configuration.

## 4. Target Audience
- **Primary Persona:** Professional TikTok Affiliate marketers focused on high-volume scaling who prefer manual control over account safety.
- **Pilot Niche:** Fashion.

## 5. Functional Requirements (MVP)

### FR-1: Human-Assisted Data Input (Excel/CSV)
- **Description:** System must process product data exported from Kalodata by a human.
- **Priority:** Must Have
- **Acceptance Criteria:** Successfully parses Excel/CSV files containing product metadata (Name, Price, Commission, Assets, etc.).

### FR-2: Gemini-Powered Scripting Engine
- **Description:** Use Gemini 2.0 to generate unique, persuasive scripts for each product in the input file.
- **Priority:** Must Have
- **Acceptance Criteria:** Generates unique script variations per product based on Kalodata insights.

### FR-3: Multi-Niche Architecture
- **Description:** Support niche-specific configurations via the `config/niches/` directory.
- **Priority:** Must Have
- **Acceptance Criteria:** YAML-based configuration for different product categories.

### FR-4: Automated Video Assembly
- **Description:** Automatically combine product assets with AI scripts and dynamic visual elements.
- **Priority:** Must Have
- **Acceptance Criteria:** Produces ready-to-post videos in a dedicated output directory.

### FR-5: Manual Posting Flow
- **Description:** System prepares metadata and video files for easy manual upload by a human.
- **Priority:** Must Have
- **Acceptance Criteria:** Organizes videos and captions in a way that minimizes manual effort for the human poster.

### NFR-1: Asset Management
- **Description:** Efficient handling of local assets (images/videos) provided in the input file or downloaded based on URLs.
- **Priority:** Must Have

### NFR-2: Containerization (Docker)
- **Description:** Project must be fully dockerized with a standard `dc.sh` wrapper.
- **Priority:** Must Have

### NFR-3: Observability
- **Description:** Logs for the generation process (CSV -> Script -> Render).
- **Priority:** Must Have

## 7. Success Metrics
- **Efficiency:** Reduce manual video creation time by 80%.
- **Output Quality:** High engagement potential scripts generated by Gemini.
- **Security:** Zero automated posting related bans (since posting is manual).

## 8. Out of Scope (Post-MVP)
- Fully autonomous scraping (re-evaluating later).
- Fully autonomous posting (re-evaluating later).
- Automated comment management.
